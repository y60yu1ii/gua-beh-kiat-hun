# Requirements: 通書來源新增暨計算精準化

## User Stories

### RS-1: 典籍來源擴充
作為用戶，我希望系統標明所有計算依據的典籍名稱與原文出處，以便查證信任。

### RS-2: 計算邏輯透明化
作為用戶，我希望每項吉凶判斷都有典籍原文依據，而非僅口訣或臆測。

### RS-3: 精準度驗證
作為開發者，需要逐一核對現有 JS 實作與典籍原文是否完全吻合。

## Acceptance Criteria

| ID | Criterion | Verification Method |
|----|-----------|---------------------|
| AC-1 | 新增至少 3 部典籍來源至典籍庫說明區 | UI 截圖確認 |
| AC-2 | 每項煞氣判斷含典籍原文引述（≥10條） | 代碼注釋審查 |
| AC-3 | 十二建除黃黑道歸屬明確標注 | 對照《協紀辨方書》原文 |
| AC-4 | 伏吟/反吟煞（含年柱/歲運）已實現或明確標記「未實現」 | 代碼審查 |
| AC-5 | 現有五部典籍計算邏輯與典籍原文核對無誤 | 逐條比對記錄 |
| AC-6 | 現有 JS 無語義錯誤，頁面正常載入 | browser_navigate 確認 |

## Scope

- 僅涉及 `index.html` 修改
- CDN 依賴（lunar-javascript）保持不變
- 不改變現有 UI 佈局
- 新增典籍說明文字至典籍庫卡片
